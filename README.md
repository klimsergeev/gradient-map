# Gradient Map — обложки спектакля с автоперекраской

Экспериментальный веб-демо: витрина спектакля «Актриса», где фон баннера с декоративными SVG-полосками автоматически адаптирует цветовую гамму под выбранную обложку.

## Что это

Страница с фиксированным макетом 1920px, адаптируется под ширину экрана через скейлинг (на узких экранах сжимается пропорционально, не ломая вёрстку). Содержит:

- **Баннер** с текстом, кнопкой покупки билетов и обложкой спектакля. Фон баннера — градиент с 11 декоративными SVG-полосками поверх.
- **Карусель из 6 пресетов** — готовые обложки разной степени «всратости» (от строгой до китча).
- **Drag-n-drop** — можно перетащить свой файл (jpeg/png/webp, до 5 МБ). Изображение автоматически сжимается в webp ≤400 КБ, кадрируется под баннер.
- **Debug-секция «Основные цвета изображения»** — показывает 5 доминирующих цветов текущей обложки кружками (временная, для визуальной отладки).
- **Переключатели гармонии и распределения** — меняют логику перекраски полосок (см. ниже).

## Как работает автоподбор цветов

При каждой смене картинки (клик по пресету или drop файла) происходит:

1. **Извлечение палитры**. Из картинки вычисляются 5 доминирующих цветов — тех, у которых больше всего близких по оттенку пикселей. Под капотом используется библиотека Color Thief (MMCQ — Modified Median Cut Quantization). Результат сортируется по площади: первый цвет самый доминирующий.
2. **Фильтр мусорных цветов**. Отсекаются почти-чёрные, почти-белые и низконасыщенные оттенки, чтобы в топ-5 не попадали пиксели фона или неинтересный серый. Если после фильтра остаётся меньше 5 цветов (например, картинка ч/б) — возвращаются первые 5 нефильтрованных.
3. **Кеш палитры**. Для пресетов палитра сохраняется в localStorage, чтобы повторные клики были мгновенными. Для пользовательских загрузок кеш не используется (каждый drop пересчитывается с нуля). При перезагрузке страницы кеш обнуляется.
4. **Перекраска полосок** (если выбрана не-Brand гармония). Все 11 градиентов перекрашиваются согласно выбранной гармонии.

## Гармонии

**Оригинал** — исходный рыже-коричневый градиент полосок, без перекраски. Режим по умолчанию.

**Аналоговая** — для каждой полоски её базовый цвет сдвигается на ±25° по кругу hue, давая три родственных оттенка на 3-стоп градиенты и ±15° на 2-стоп. Мягко, не контрастно, сохраняет общее настроение картинки.

**Триада** — классическая триада 120°: базовый цвет плюс два спутника, разнесённых на 120° по кругу. Максимальный контраст. Для каждой полоски случайно (но детерминированно) выбирается один из трёх вариантов порядка стопов. 2-стоп полоски получают сжатую триаду ±60°.

**Комплиментарная** — база + противоположный цвет через 180° по кругу hue. Максимальный двуцветный контраст. 3-стоп полоски получают один из двух симметричных вариантов: `[base, comp, base]` или `[comp, base, comp]`. 2-стоп — прямой переход `[base, comp]`.

**Монохромная** — один hue, варьируется только светлота (L ±20%) и слегка насыщенность (S ±10%, синхронно: тёмные стопы менее насыщенные — как тени в реальности). Jitter работает по светлоте (±3%), а не по hue — соседние полоски одного базового цвета слегка различаются по яркости. Самая сдержанная гармония.

**Квадратичная** — 4 цвета через 90° (квадрат на цветовом круге). Для 3-стоп полосок берутся 3 подряд вершины квадрата, стартовая вершина выбирается случайно (детерминированно) для каждой полоски. 2-стоп полоски получают смежные вершины (четверть квадрата, не противоположные — это отличает от комплиментарной).

**Разделённая комплиментарная** — модификация комплиментарной: вместо точного комплемента берутся два соседних к нему цвета (±30° от 180° = 150° и 210°). Мягче, чем чистый комплемент, но с сохранением контрастности. 2-стоп полоски получают только пару split-цветов без базы.

## Распределения

При перекраске нужно «раздать» 5 базовых цветов на 11 полосок. Доступно два режима (переключатель виден, когда выбрана не-Brand гармония):

**Mix** (по умолчанию) — интерливинг: доминирующий цвет разнесён по разным слоям, визуально удерживает композицию сверху донизу. Карта `[0,1,2,3,4,0,1,2,0,3,1]` (индекс = z-слой полоски минус 1).

**Zones** — зональное распределение: смежные полоски одного базового цвета. Карта `[0,0,0,1,1,1,2,2,3,3,4]`. Даёт более выраженные «цветовые зоны» вместо равномерной смеси.

## Дополнительные правила

- **Ч/б фолбэк**. Если палитра целиком серая (например, 2-bw пресет), гармония не применяет hue-сдвиги и оставляет стопы серыми с вариацией по светлоте. Полоски сливаются с фоном — это временное поведение, в будущем может быть заменено.
- **Прозрачные стопы**. Стопы с `stop-opacity="0"` (они есть почти в каждой полоске — тающие края) наследуют hue ближайшего непрозрачного соседа, чтобы переход к прозрачности не выдавал чужой оттенок.
- **S/L clamping**. После вычисления каждый цвет насильно прижимается в диапазон S ∈ [45%, 90%], L ∈ [25%, 65%]. Это нужно для читаемости на тёмном фоне баннера — пересвеченные и беспросветные палитры не «растворятся» и не «выжгут».
- **Jitter**. Для каждой полоски все её стопы сдвигаются целиком на случайный (seeded) угол. Для поворотных гармоний (аналоговая, триада, комплиментарная, квадратичная, разд. комплиментарная) — jitter ±10° по hue, соседи с одной base различаются ≥5°. Для монохромной — jitter ±3% по светлоте (hue фиксирован), соседи различаются ≥1.5%.
- **Seed-стабильность**. Одна и та же палитра всегда даёт один и тот же результат перекраски — и для тех же полосок, и в том же распределении. Seed — простой хеш строкового представления палитры. Это значит: переключил пресет, вернулся обратно — полоски идентичны.
- **Восстановление из Brand**. При первом инжекте полосок их оригинальные цвета сохраняются в `data-original-color` атрибутах. Переключение на Brand читает эти атрибуты и возвращает всё как было, без перезагрузки SVG.

## Сохранение состояния

Между сессиями в localStorage хранятся:
- выбранный пресет (при reload очищается, страница открывается с дефолтным `1-aronova`)
- режим гармонии (`brand` / `analogous` / `triadic` / `complementary` / `monochromatic` / `tetradic` / `splitComplementary` — сохраняется между перезагрузками)
- режим распределения (`mix` / `zones` — сохраняется между перезагрузками)
- кеш палитр для пресетов (очищается при reload)

Пользовательские загрузки не переживают reload — при перезагрузке страница возвращается к дефолтному пресету.

## Технический стек

Vanilla HTML + CSS + JavaScript (ES-модули), без сборщиков и npm-зависимостей. Единственная внешняя зависимость — Color Thief через CDN unpkg. Хостинг — GitHub Pages.
